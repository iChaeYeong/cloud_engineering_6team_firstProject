ERD (Entity-Relationship Diagram)

계좌이체 시스템 — 클라우드 엔지니어링 6조 1차 프로젝트

user_info 사용자
컬럼명타입제약설명
user_id VARCHAR(20) PK 로그인 아이디
user_nm VARCHAR(20) NOT NULL 사용자 이름
pwd VARCHAR(255) NOT NULL BCrypt 해시 비밀번호
accounts_info 계좌
컬럼명타입제약설명
account_no VARCHAR(30) PK 계좌번호
user_id VARCHAR(20) FK NOT NULL 소유 사용자 (→ user_info)
balance INT DEFAULT 0 잔액 (원)
account_nm VARCHAR(30) NOT NULL 계좌 별칭
account_stat VARCHAR(20) NOT NULL 계좌 상태 (예: 정상, 정지)
transaction_history 거래내역
컬럼명타입제약설명
tran_id INT PK AUTO_INCREMENT 거래 고유번호
frm_account_no VARCHAR(30) FK NOT NULL 출금 계좌 (→ accounts_info)
to_account_no VARCHAR(30) FK NOT NULL 입금 계좌 (→ accounts_info)
tran_amt INT NOT NULL 이체 금액 (원)
tran_dt DATETIME DEFAULT NOW() 거래 일시

테이블 관계

관계부모 테이블자식 테이블참조 키카디널리티
사용자 → 계좌 user_info accounts_info user_id 1 : N
계좌 → 거래(출금) accounts_info transaction_history frm_account_no 1 : N

DDL (schema.sql)

CREATE TABLE user_info (
    user_id     VARCHAR(20)  PRIMARY KEY,
    user_nm     VARCHAR(20)  NOT NULL,
    pwd         VARCHAR(255) NOT NULL
);

CREATE TABLE accounts_info (
    account_no   VARCHAR(30)  PRIMARY KEY,
    user_id      VARCHAR(20)  NOT NULL,
    balance      INT          DEFAULT 0,
    account_nm   VARCHAR(30)  NOT NULL,
    account_stat VARCHAR(20)  NOT NULL,
    FOREIGN KEY (user_id) REFERENCES user_info (user_id)
);

CREATE TABLE transaction_history (
    tran_id        INT          PRIMARY KEY AUTO_INCREMENT,
    frm_account_no VARCHAR(30)  NOT NULL,  -- 출금 계좌
    to_account_no  VARCHAR(30)  NOT NULL,  -- 입금 계좌
    tran_amt       INT          NOT NULL,
    tran_dt        DATETIME     DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (frm_account_no) REFERENCES accounts_info (account_no),
    FOREIGN KEY (to_account_no)  REFERENCES accounts_info (account_no)
);
