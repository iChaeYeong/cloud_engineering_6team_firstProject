SCR-01

로그인

/login
URL
/login
접근 권한
비로그인 (기본 진입점)
인증 방식
HTTP Session (쿠키)
API
POST /auth/login

입력 요소

컴포넌트타입필수설명
아이디text input필수user_info.user_id에 매핑
비밀번호password input필수BCrypt 해시 비교, Enter 키로 제출 가능
로그인 버튼button—처리 중 비활성화, 스피너 텍스트 표시
회원가입 링크Link—/signup 이동

동작 흐름

입력 검증 (빈 값) → POST /auth/login → 세션 저장
  ├── 성공: /accounts 리디렉트
  └── 실패: 에러 메시지 표시 ("아이디 또는 비밀번호가 올바르지 않습니다.")
SCR-02

회원가입

/signup
URL
/signup
접근 권한
비로그인
API
POST /auth/register

입력 요소

컴포넌트타입필수설명
아이디text input필수user_id (VARCHAR 20)
이름text input필수user_nm (VARCHAR 20)
비밀번호password input필수저장 시 BCrypt 해시 적용
회원가입 버튼button—처리 중 비활성화

동작 흐름

POST /auth/register
  ├── 성공: /login 리디렉트 (성공 메시지)
  └── 실패: 에러 메시지 표시 (중복 아이디 등)
SCR-03

계좌 목록

/accounts
URL
/accounts
접근 권한
로그인 필요
API
GET /accounts/list

표시 요소

요소설명
총 보유자산계좌 잔액 합산, 계좌가 1개 이상일 때만 표시
계좌 카드 목록계좌명 / 계좌번호 / 상태 배지 / 잔액
내역 버튼/transactions?account={accountNo} 이동
이체 버튼/transfer?from={accountNo} 이동

동작 흐름

페이지 마운트 → GET /accounts/list
  ├── 성공: 계좌 목록 렌더링
  └── 인증 실패 (401): /login 리디렉트
SCR-04

계좌 이체

/transfer
URL
/transfer?from={accountNo}
접근 권한
로그인 필요
API
POST /accounts/transfer

입력 요소

컴포넌트타입필수설명
출금 계좌select필수내 계좌 목록 드롭다운, 현재 잔액 표시
입금 계좌번호text input필수수취인 계좌번호 직접 입력
이체 금액number input필수양수, tabular-nums 폰트
이체하기 버튼button—처리 중 비활성화
취소 링크Link—/accounts 이동

동작 흐름

입력 검증 (빈 값 / 금액 유효성) → POST /accounts/transfer
  ├── 성공: /accounts 리디렉트
  └── 실패: 에러 메시지 표시 (잔액 부족 등)
SCR-05

거래 내역

/transactions
URL
/transactions?account={accountNo}
접근 권한
로그인 필요
API
GET /transactions/history?accountNo=

표시 요소

컬럼설명
거래 일시tran_dt (DATETIME)
출금 계좌frm_account_no
입금 계좌to_account_no
거래 금액tran_amt (천단위 콤마)

동작 흐름

쿼리파라미터 account 읽기 → GET /transactions/history?accountNo={account}
  ├── 성공: 거래 내역 테이블 렌더링
  └── 내역 없음: "거래 내역이 없습니다." 메시지
